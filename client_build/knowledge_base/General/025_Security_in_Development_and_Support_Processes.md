## Security in Development and Support Processes

1. Development Rules: Establish and apply rules for the development of software and systems within Arabian Mills.
2. Change Control: Control changes to systems within the development lifecycle using formal change control procedures.
3. Platform Changes: Review and test business-critical applications when operating platforms are changed to ensure no adverse impact on Arabian Mills operations or security.
4. Software Modifications: Discourage modifications to software packages, limit changes to necessary ones, and strictly control all changes.
5. Secure Systems Principles: Establish, document, maintain, and apply principles for secure systems to any information system implementation efforts.
6. Development Environments: Establish and protect secure development environments for system development and integration efforts covering the entire system development lifecycle.
7. Outsourced Development: Supervise and monitor the activity of outsourced system development.
8. Security Requirements: Include information security-related requirements in the requirements for new information systems or enhancements to existing systems.
9. Security Testing: Carry out testing of security functionality during development.
10. Acceptance Testing: Establish acceptance testing programs and related criteria for new information systems, upgrades, and new versions.
11. Test Data: Select, protect, and control test data carefully.
12. Secure Coding Practices: Develop and implement secure coding practices, using trusted and licensed sources for software development tools and libraries.
13. Integration Practices: Follow secure practices for integration between different software.
14. Penetration Testing: Conduct application penetration tests before go-live.
