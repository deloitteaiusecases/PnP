## Security Event Logging and Monitoring

1. Requirements: Define, document, approve, and implement cybersecurity requirements for security event logs and monitoring.
2. Event Logs: Produce, keep, and regularly review event logs recording user activities, exceptions, faults, and information security events.
3. Critical Infrastructure: Activate appropriate security event logging on critical infrastructure and privileged user accounts.
4. Methodology: Use an appropriate methodology for event log collection and continuous monitoring.
5. Retention Period: Retain cybersecurity event logs for a minimum of 12 months.
6. Periodic Review: Review cybersecurity requirements for event logs and monitoring management periodically.
7. Log Protection: Protect logging facilities and log information against tampering and unauthorized access.
8. Administrator Activities: Log system administrator and system operator activities, protect the logs, and review them regularly. System administrators must not have permission to erase or deactivate logs of their own activities.
9. Log Safeguarding: Use real-time copying of logs to a system outside the control of a system administrator or operator to safeguard logs.
10. Clock Synchronization: Synchronize the clocks of all relevant information processing systems and devices within Arabian Mills to a single reference time source.
11. Standard Reference Time: Define and configure a standard reference time for use within Arabian Mills.
