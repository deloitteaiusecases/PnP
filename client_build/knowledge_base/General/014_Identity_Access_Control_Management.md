## Identity & Access Control Management

1. Requirements: Define, document, approve, and implement cybersecurity requirements for identity and access management based on business and information security requirements.
2. Authorization Principles: Implement user authorization based on identity and access control principles: Need-to-Know, Need-to-Use, Least Privilege, and Segregation of Duties.
3. Access Control Rules: Ensure asset owners determine appropriate access control rules, access rights, and restrictions for specific user roles towards their assets.
4. Remote Access: Restrict remote access to business-critical applications.
5. Network Access: Provide users with access to the network and network services they are specifically authorized to use.
6. User Registration: Implement a formal user registration and de-registration process to enable the assignment of access rights.
7. Access Provisioning: Implement a formal user access provisioning process to assign or revoke access rights for all user types to all systems and services.
8. Privileged Access: Restrict and control the allocation and use of privileged access rights.
9. Secret Authentication: Control the allocation of secret authentication (password) information through a formal management process.
10. Access Review: Ensure asset owners review users' access rights at regular intervals.
11. Termination: Remove access rights of all employees and external party users to information and information processing facilities upon termination of their employment, contract, or agreement, or adjust upon change.
12. Practices: Require users to follow Arabian Mills' practices in the use of secret authentication information.
13. Access Restriction: Restrict access to information and application system functions in accordance with access control guidelines.
14. Generic User IDs: Restrict the use of generic user IDs.
15. Secure Log-On: Where required, control access to systems and applications by a secure log-on procedure.
16. Password Management: Ensure password management systems are interactive and ensure quality passwords.
17. Utility Programs: Restrict and tightly control the use of utility programs that might be capable of overriding system and application controls.
18. Source Code Access: Restrict access to program source code.
19. User Authentication: Implement user authentication based on username and password.
20. Multi-Factor Authentication: Implement multi-factor authentication for remote access based on risk assessment and asset owners' requirements.
21. Privileged Access Restriction: Restrict privileged access.
22. Periodic Review: Review the implementation of cybersecurity requirements for identity and access management periodically.
