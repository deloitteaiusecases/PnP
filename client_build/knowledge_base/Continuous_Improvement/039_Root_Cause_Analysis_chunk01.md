## Root Cause Analysis

Introduction and Background
Root Cause Analysis is a systematic method used to identify the origin of problems or failures and determine corrective actions that prevent recurrence. Without structured RCA, the same issues are likely to repeat, leading to:
 Increased reactive maintenance
 Higher operational risk
 Inefficient use of manpower and spares
This section establishes a formal RCA system using global best practices, including:
 Clear RCA triggers based on KPIs
 Use of structured tools
 Defined roles and accountability
 Integration with the SAP PM and review systems
Commonly Used RCA Tools

| Tool | Purpose | How It Is Used |
| --- | --- | --- |
| 5 Whys | Quickly find the root of simple problems | Ask "Why?" iteratively (usually five times) until the true cause is uncovered |
| Fishbone Diagram (Ishikawa) | Visualize contributing factors across categories (e.g., People, Methods, Machine) | Used during brainstorming to map all potential causes |
| Pareto Chart | Identify the most frequent failure modes or causes | Based on failure frequency or cost; helps prioritize what to address |

Procedure – Root Cause Analysis

| S. No. | Procedure Description | Responsibility | Frequency |
| --- | --- | --- | --- |
| 1 | **Define RCA Trigger Criteria**<br>• Establish thresholds for when RCA is mandatory (e.g., unplanned downtime > 4 hours, repeat failure within 1 month, critical safety event). | Maintenance Director | One-time / Annual Review |
| 2 | **Develop RCA Template and Checklist**<br>• Design standard forms for RCA documentation including problem description, timeline, tool used, team members, and actions. | Maintenance Manager | One-time / As Updated |
| 3 | **Train Staff on RCA Tools**<br>• Conduct workshops for engineers and supervisors on 5 Whys, Fishbone, and Pareto methods. | Maintenance Manager | Annual / New Joiners |
| 4 | **Initiate RCA Based on Trigger**<br>• When triggered, assign a facilitator and team to conduct the RCA. | Maintenance Manager | As Needed |
| 5 | **Conduct RCA Session**<br>• Facilitate structured analysis using approved tools, document all findings, and identify true root causes. | Assigned RCA Team | As Needed |
| 6 | **Define and Assign Corrective Actions**<br>• Translate causes into specific, measurable actions with deadlines and owners. | RCA Team | As Needed |
| 7 | **Implement and Monitor Corrective Actions**<br>• Track action progress using action tracker or SAP PM work order system. | Maintenance Supervisor | Ongoing |
| 8 | **Review Effectiveness**<br>• After a defined period, verify whether the corrective actions prevented recurrence. | Maintenance Director | Quarterly |
| 9 | **Report RCA Findings in Monthly Review**<br>• Include major RCA cases and lessons learned in KPI review meetings. | Maintenance Manager | Monthly |
| 10 | **Maintain RCA Library**<br>• Archive all RCA reports for future reference and knowledge sharing. | SAP PM Administrator | Ongoing |

Example of a RCA Template

| Section | Details to be Filled | Information |
| --- | --- | --- |
| 1. Basic Information | Equipment Name, Asset ID, Department, Date of RCA, Facilitator, Participants |  |
| 2. Problem Description | Clear, concise description of the failure, event, or recurring issue |  |
| 3. Timeline of Events | Chronological sequence leading up to and during the incident |  |
| 4. Immediate Causes | Initial visible failures or malfunctions (symptoms) |  |
| 5. Root Cause Analysis Method Used | e.g., 5 Whys, Fishbone Diagram, Pareto |  |
| 6. Contributing Factors (Fishbone Categories) | Man, Machine, Method, Material, Measurement, Environment (brief assessment under each) |  |
| 7. Root Cause(s) Identified | List of true root causes identified through the selected tool |  |
| 8. Corrective and Preventive Actions | List each action, whether it's corrective or preventive, with rationale |  |
| 9. Action Owner and Deadline | Name, designation of person responsible, and target completion date for each action |  |
| 10. Verification of Effectiveness | Describe method of verification (e.g., no recurrence in 30 days, performance trend improved) |  |
| 11. Lessons Learned | What should be shared with other teams to avoid recurrence elsewhere |  |
| 12. Review and Approval | Name and signature of RCA owner, approver, and review date |  |

Roles and Responsibilities – Root Cause Analysis

| Role | Responsibilities |
| --- | --- |
| Technician | • - Provide accurate feedback about failure symptoms and timeline.<br>• - Participate in RCA sessions as needed. |
| Maintenance Supervisor | • - Trigger and facilitate RCAs at the shop floor level.<br>• - Ensure participation of relevant team members. |
| Maintenance Manager | • - Lead RCA sessions.<br>• - Select and apply suitable RCA tools.<br>• - Define corrective actions. |
| Maintenance Director | • - Approve RCA outcomes and ensure alignment with strategic goals.<br>• - Review effectiveness during monthly reviews. |
| Planner | • - Record corrective actions and follow up deadlines.<br>• - Raise WOs for systemic improvements in SAP. |
| SAP PM Administrator | • - Store RCA documents in the SAP document management system.<br>• - Support linking RCA outcomes to SAP master data. |
| Safety Officer | • - Support RCA for safety-related incidents.<br>• - Ensure integration with incident investigation and HSE system. |
